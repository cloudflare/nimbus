The sidebar comes from one of two sources: the filesystem (zero config) or an explicit `items` array in your Nimbus config. Both live under `sidebar` in `astro.config.ts`.

## Autogenerate from the filesystem

With no `items`, the rail mirrors `src/content/docs/` — top-level files become links, directories become groups, and a directory's `index.mdx` becomes the group's landing page. Order and labels come from [frontmatter](/writing/frontmatter):

```yaml
sidebar:
  order: 2
  label: Quickstart
  badge: New
```

## Explicit items

For full control, list items yourself. Each can be a bare slug, a labeled group that autogenerates from a directory or collection, a manual group, or a link:

```ts title="astro.config.ts"
sidebar: {
  items: [
    "introduction",
    "installation",
    { label: "Writing", autogenerate: { directory: "writing" } },
    { label: "Components", autogenerate: { collection: "components" } },
    { label: "GitHub", link: "https://github.com/you/repo" },
  ],
}
```

- **Bare slug** (`"introduction"`) — a single page; order follows array position.
- **`autogenerate: { directory }`** — build a group's children from a subfolder.
- **`autogenerate: { collection }`** — pull in another collection (e.g. `components`), mounted at its own prefix.
- **`items: [...]`** — a hand-built nested group.

## Order, badges, collapse

Within an autogenerated group, child order comes from each page's `sidebar.order` (alphabetical fallback). Per-group options:

```ts
{ label: "Reference", autogenerate: { directory: "reference" }, collapsed: true, badge: "API" }
```

Site-wide collapse and scope knobs:

```ts
sidebar: {
  items: [/* … */],
  defaultCollapsed: true,   // every group starts closed (active group still opens)
  scope: "section",         // rail shows only the current top-level section
}
```

`scope: "section"` pairs with the header section tabs for large sites; the default `"full"` shows the whole tree on every page.

## Collapse a section to one link

Set `sidebar.hideChildren: true` on a directory's `index.mdx` to render the whole section as a single link to that page.
